# Contributing to GattyWorks

Thanks for your interest in contributing. These are the default contribution guidelines for every repository in the [GattyWorks](https://github.com/gattyworks) organization. A repository may add its own CONTRIBUTING file that takes precedence.

By participating, you agree to our [Code of Conduct](CODE_OF_CONDUCT.md).

## Ways to help

- **Report a bug:** open a [bug report](https://github.com/gattyworks/.github/issues/new/choose) with steps to reproduce.
- **Request a feature:** open a [feature request](https://github.com/gattyworks/.github/issues/new/choose) and explain the problem it solves.
- **Send a fix:** small, focused pull requests are the easiest to review and merge.

## Development workflow

We work in short-lived branches and ship through pull requests. We never push directly to `main`.

1. Fork the repository, or create a branch if you have write access.
2. Create a branch with a descriptive name, for example `fix/login-redirect`.
3. Make your change. Keep it focused on one thing.
4. Follow the existing code style. Run the linters and tests if the repo has them.
5. Write a clear commit message that explains the why, not just the what.
6. Open a pull request against `main` and fill in the template.
7. A maintainer reviews it. Address feedback, then we merge.

## Pull request checklist

- The change is scoped to a single concern.
- Tests and linters pass locally.
- Documentation is updated if behavior changed.
- No secrets, keys, or credentials are committed.
- The pull request description explains the change and links any related issue.

## Commit messages

- Use the imperative mood: "Add", "Fix", "Refactor", not "Added" or "Fixes".
- Keep the summary under about 72 characters.
- Add a body when the change needs context.
